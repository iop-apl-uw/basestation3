FM VBD Bias Plot Help

This plot is a FlightModel mission-summary plot, not a per-dive science plot. It shows, for each dive processed by FlightModel, how much the vehicle's apparent buoyancy-engine volume (vbdbias) differs from the assumed maximum volume (volmax) - a sanity check on FlightModel's core buoyancy assumption over the course of a mission.



Per-dive volume change

The per-dive vbdbias value FlightModel solved for that dive (volume decrease, in cc, from the assumed volmax). Dives where no solution could be found are marked separately (see "No solution" below) rather than plotted here.



Median volume change

A running median of the per-dive vbdbias values over the most recent dives (window size controlled by the vbdbias_filter parameter), used to smooth out per-dive noise and show the underlying trend.



min w_rms*10 (cm/s)

The minimum vertical-velocity RMS misfit found while solving for that dive's vbdbias, scaled by 10 so it's visible on the same axis as the volume traces - a lower value indicates a better-constrained solution for that dive.



No solution

Marks dives (at y=0) where FlightModel could not resolve a vbdbias value at all, for example due to insufficient or noisy dive data.



Implied C_VBD (AD counts)

An optional secondary y-axis, only shown when the pilot has declared a reference C_VBD value. Converts the volume-change traces into the equivalent buoyancy pump (VBD) AD-count offset, using the glider's VBD_CNV conversion factor - helps judge whether the reference C_VBD (the buoyancy engine's assumed neutral point) needs to be re-tared.
