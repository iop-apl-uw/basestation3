WKB Sampling Schedule Plot Help

This plot proposes a CTD sampling interval that is uniform in WKB-stretched (buoyancy-frequency-normalized) depth, rather than uniform in raw depth, derived from the same trailing window of dives (--wkb_dives_back) as the companion WKB Buoyancy Frequency plot. Separate dive and climb subplots are shown, each as sampling interval (seconds) vs. time into the dive. The screenshots below are from SG272's dive 300.



Toggle Buttons

The All/Current/Option N buttons above the plot show or hide that group's traces (in both subplots at once) - click a button once to hide its group, click again to show it. Each group can be toggled independently, so any combination can be shown together; the rest of this page shows just Current and Option 0 for clarity, since all three options together can be hard to tell apart. Showing an Option N group also reveals its proposed ct = { ... } text below the plot (see "Proposed Schedule Text" below) - the two are always shown/hidden together. Copy schedule copies whichever proposed schedule text is currently visible to the clipboard, and only appears once a schedule is showing.



Current Sampling

The thick gray current sampling trace is the sampling interval actually used for the dive, as currently configured.



WKB Sampling

The thin wkb sampling, option N traces are the proposed WKB-stretched sampling intervals, one per candidate relative-buoyancy-frequency option evaluated (--wkb_relative_n) - noisier than the current sampling trace since they aren't bucket-averaged (see "Points" below). Each option has its own color, consistent across the dive and climb subplots, the legend, and the proposed schedule text: option 0 is red, option 1 is black, option 2 is blue.



Points

The thicker N points traces highlight, for each option, the sampling interval that would be used if sampling at a fixed 1-m vertical resolution (bucket-averaged from the thin WKB sampling trace above, same color), annotated with the resulting total point count for that option. The image below shows Current (gray) and Option 0 (red) together - the thin, noisier red line is wkb sampling, option 0; the thick red line with the 439 points label is that option's bucket-averaged rate.



Depth Reference Lines

The vertical dashed blue reference lines and labels (also visible in the image above) mark the dive time at which the glider crosses each labeled depth, for orientation between the depth and time axes.



Proposed Schedule Text

Below the plot, a permanent note reports the dive's actual CT sensor type and mount (truck/scicon/logger - see WkbConfigText.ct_mount). Selecting Current or an Option N button (see "Toggle Buttons" above) shows that group's proposed sampling schedule as ct = { depth,seconds ... } text, in the same format as scicon.sch, laid out in two columns to keep it compact. The first line is a comment (/ ..., ignored by validate.sciconsch) identifying which schedule it is - either Current, or the option's point-count multiplier (e.g. / 0.8 * points for the 0.8 in --wkb_relative_n). Depth buckets deeper than the dive actually reached are omitted. Use the Copy schedule button (see "Toggle Buttons" above) to copy this text - including the comment line, but not the two-column layout, which is display-only - to the clipboard.



Options

These options configure the WKB computation shared by this plot and the companion WKB Buoyancy Frequency plot:

  • --wkb_dives_back: How many dives back (inclusive of the current dive) to include.
  • --wkb_buckets_depths: Comma-separated depth bucket boundaries (m), e.g. 100,200,300,400,500,600,700,800,1200. The first value also marks the bottom of the fixed-rate top zone (see --wkb_top_sampling_rate).
  • --wkb_relative_n: Comma-separated candidate point-count multipliers (relative to the current sampling), e.g. 0.8,1,1.5 - one wkb sampling, option N trace is proposed per value.
  • --wkb_dz: Depth bin size (m) used when gridding dives.
  • --wkb_z_max: Maximum depth (m) used when gridding dives.
  • --wkb_top_sampling_rate: Fixed sampling interval (s) imposed above the first depth bucket.
  • --wkb_buoyfreq/--no-wkb_buoyfreq: Whether to also generate the companion WKB Buoyancy Frequency plot (default on).
