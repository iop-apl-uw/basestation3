This plot summarizes file transfer activity per call session, parsed from comm.log. Each point on the x-axis is one call (session); the secondary top x-axis maps calls to dive number.

Traces

  • (bytes) traces (right Bytes axis, solid lines) show total bytes transferred per call, broken out by file type (e.g. compressed log/data/eng files, kkyy files, etc).
  • (count) traces (left Count axis, dashed lines) show the number of files of each type transferred per call.
  • (secs) traces (Seconds axis, dotted lines) show time spent transferring each file type per call.

Buttons

The buttons below the plot toggle which trace group is shown: All, Sizes, Counts, Times, Totals, or an individual file type. Toggling a button shows that group and hides the rest (shift/legend click behaves as usual for individual traces).

CRC errors (failed/retried packets) reported in comm.log are reflected in the underlying data but are not broken out as a separate trace on this plot.
